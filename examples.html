Puppy Examples

Welcome to Puppy! This page demonstrates a few features that you might find useful while crafting your next awesome prototype.

Front Matter Headers

YAML data in the header of your content pages are exposed via the current_page variable. For instance, in this template I've defined the following:


names: ['Tito', 'Jared', 'Mike']

I can now iterate over these names using a Twig for loop over current_page.names:

  • Item # 1: Tito
  • Item # 2: Jared
  • Item # 3: Mike

Sitemap

You can access front-matter headers from all of your content pages using the sitemap property. Pages are indexed by file name relative to the ./src/content directory, so you can see the headers for this page by looking at sitemap.examples: 

object(4) {
  [title]=> 
  string(8) "Examples"
  [menu]=> 
  bool(false)
  [names]=> 
    object(3) {
    [0]=> 
    string(4) "Tito"
    [1]=> 
    string(5) "Jared"
    [2]=> 
    string(4) "Mike"
  }
  [path]=> 
  string(14) "/examples.html"
}
The sitemap object stores pages in sub-directories recursively which can be accessed using dot notation. For instance, an `about/index.html` page would be available at `sitemap.about.index`.

Data Fixtures

If you have any external data that you'd like to reuse across several templates, you can add them as JSON files in the ./data directory. This list of Upstatement office locations is pulled from the ./data locations file, and exposed via the data.locations template variable:

  • Office #1: 319 A Street Boston, MA 02210
  • Office #2: 745 Atlantic Ave, Floor 3 Boston, MA 02111
  • Office #3: 133 Portland Street, 5th Floor Boston, MA 02114

This is handy for doing things like mocking content from dynamic sources (e.g. WordPress, APIs, etc.), and helps you keep your templates DRY by eliminating duplicate markup for repeated content blocks.

ES2015, ES2016 Support

Support for modern JavaScript Syntax and features can be added by selecting ES2015+ during yo puppy. This adds Browserify and Babel to your puppy project. This means you'll need to import modules in a new way.

Import with NPM:

If possible, modules should be imported with npm, not bower.

  1. Check if the module exists as a CommonJS module by searching npm.
  2. Install the module with npm install --save <module-name>
  3. Where the module is needed use the import statement. For example, import $ from 'jQuery';

Import with Bower:

If the dependency is unavailable in NPM, you may have to use bower.

  1. bower install --save <your-dependency>
  2. Add a <script> tag referencing the vendor script in layouts/base.twig:
<!-- build:js /static/js/vendor.js -->
<script type="text/javascript" src="/bower_components/scrollmagic/scrollmagic/minified/ScrollMagic.min.js"></script>
<!-- endbuild -->
  1. Where you need to call the dependency, you'll need to access it from the window namespace: ex. window.ScrollMagic Within a module you could assign the dependency to a constant: ex. const Modernizr = window.Modernizr;.

Markdown Support!

Markdown support is available.

The following content is pure Markdown...

Markdown is a way to style text on the web. You control the display of the document; formatting words as bold or italic, adding images, and creating lists are just a few of the things we can do with Markdown. Mostly, Markdown is just regular text with a few non-alphabetic characters thrown in, like # or *.
